# TypeDefinition 跨表引用示例

本目录用于展示一份足够完整、可直接复制的类型定义模板。结论是：全部内容集中在一个 `TypeDefinition.xlsx` 中，通过五个分表覆盖类型、约束和跨表引用示例；不再额外提供独立的物品表与奖励表。

注意：导出器会跳过整个 `TypeDefinition.xlsx`，因此其中本地化命名的物品与奖励分表是教学示例，不会直接生成配置。中文模板使用 `物品表` 与 `奖励表`；要实际运行，请把这两个分表复制到普通业务工作簿（如 `GameConfig.xlsx`），再新增业务 `CODE` 分表声明输出文件和平台。

教学分表第 1 行固定使用英文 camelCase 代码字段（如 `itemId`、`primaryItemId`），第 2 行的类型名按 `--locale` 使用对应语言，第 4 行使用对应语言说明；`+required()`、`+unique()` 等约束名保持固定语法。作为跨表引用源的 `itemId` 保留 `int`，引用字段则使用已有的本地化类型名，例如中文的 `物品ID`、`物品列表`。

目录内容：

- `CODE`：四列类型定义，中文模板直接包含 `物品ID`、`物品列表`、`物品列表2` 与 `物品列表3`，不重复创建近义引用类型。
- `Guide`：约束、分隔符、跨表引用参数及输出边界。
- `Examples`：可复制的类型表达式和单元格示例。
- 物品分表：被引用的数据表示例；中文名称为 `物品表`。
- 奖励分表：直接引用、引用列表、`equalLen` 与端字段分流示例；中文名称为 `奖励表`。

重新生成一份独立副本：

```powershell
python scripts/create_examples.py --output-dir my-example
```

默认只接受不存在或为空的输出目录。加 `--force` 时也只替换 `tables/TypeDefinition.xlsx`，不会触碰其他文件，也不会预先创建客户端或服务端输出目录。
